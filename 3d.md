# 3D可视化专家一年攻坚计划

## 📋 读者简介
- **当前目标**：从业务前端转向3D可视化/图形学领域，建立高竞争壁垒
- **可用时间**：周末系统学习 + 平日1小时巩固
- **终极目标**：一年后具备高级图形工程师/可视化专家竞争力

## 🧭 核心学习原则

### 1. 以战代练，项目驱动
每个阶段必须完成至少一个可展示的**完整项目**，从简单到复杂层层递进。

### 2. 原理先行，框架后置
先深入理解WebGL、数学基础，再学习Three.js等上层框架。避免成为“只会调API的框架用户”。

### 3. 周末攻坚，平日巩固
- **周末**（每天4-5小时）：系统性学习新知识，攻克难点
- **平日**（每天1小时）：复习、编码、社区互动、写技术笔记

### 4. 公开输出，建立影响力
每个阶段至少产出一篇**深度技术文章**，发布在个人博客/技术社区，积累行业声誉。

## 📅 四阶段详细规划

### 第一阶段：筑基与认知重塑（第1-3个月）
**核心目标**：打通3D图形学任督二脉，建立正确的数学和图形学认知模型。

#### 学习重点
1. **3D数学基础**（重中之重）
   - 精读《3D数学基础：图形与游戏开发》
   - 亲手推导所有公式，用JavaScript实现向量、矩阵运算库
   - 重点掌握：矩阵变换、坐标系转换、四元数基础

2. **WebGL底层原理**
   - 精读《WebGL编程指南》前8章
   - 禁用所有框架，只用原生WebGL API + Canvas
   - 理解：渲染管线、缓冲区、着色器程序、纹理映射

3. **周末项目**：纯WebGL“太阳系模拟”
   - 手动计算每个行星的模型矩阵（绕日公转+自转）
   - 理解矩阵堆栈/现代矩阵管理
   - 实现基础的光照和纹理

#### 平日任务
- 复习数学推导和WebGL代码
- 浏览Shadertoy，感受GLSL风格
- 输出文章：《从零理解模型视图投影矩阵》

### 第二阶段：引擎赋能与效率提升（第4-6个月）
**核心目标**：掌握Three.js，并用底层原理解析其工作机制。

#### 学习重点
1. **Three.js系统学习**
   - 完成Three.js Journey付费课程
   - 重点模块：相机、光照与阴影、几何体、粒子系统
   - 关键动作：每学一个高级模块，追踪其对应的WebGL调用

2. **工具链建设**
   - 学习Blender基础操作
   - 掌握glTF格式导出导入
   - 了解glTF生态系统

3. **周末项目**：“数据可视化城市”
   - 程序化生成建筑（作为数据柱状图）
   - 实现相机漫游、点击高亮、数据面板联动
   - 注重代码架构设计

#### 平日任务
- 优化可视化城市项目
- 学习Blender基础操作
- 输出文章：《Three.js Material的WebGL实现剖析》

### 第三阶段：性能攻坚与领域深化（第7-9个月）
**核心目标**：攻克工业级应用的性能瓶颈，整合实时数据。

#### 学习重点
1. **性能优化专题**
   - 合并Draw Call
   - LOD（细节层次）技术
   - 视锥体裁剪
   - GPU拾取
   - 遮挡查询

2. **着色器进阶**
   - 系统学习GLSL语法和特性
   - 实现自定义着色器效果：
     - 卡通渲染
     - 边缘光效
     - 动态水面
     - 雪地足迹

3. **行业协议与数据整合**
   - 学习MQTT协议，搭建本地EMQX环境
   - 掌握ECharts/D3.js用于2D图表补充
   - 理解物联网数据流架构

4. **周末项目**：“智慧工厂监控大屏”
   - 3D车间模型 + 设备状态展示
   - MQTT模拟数据流驱动
   - 2D图表展示产量、能耗曲线
   - 3D点击与2D面板完整联动

#### 平日任务
- 用Chrome Performance分析项目性能
- 阅读Three.js/Mapbox渲染相关源码
- 输出文章：《十万级顶点数据的加载与渲染优化实战》

### 第四阶段：整合、架构与影响力（第10-12个月）
**核心目标**：打造专家级作品，建立行业影响力，完成能力闭环。

#### 学习重点
1. **架构设计能力**
   - 为“智慧工厂”项目设计微前端/Monorepo架构
   - 实现3D场景、2D图表、数据通信模块解耦
   - 制定项目工程化规范

2. **垂直方向深挖**（三选一）
   - **数字孪生方向**：学习Cesium.js，掌握三维地理空间可视化
   - **高保真渲染方向**：研究PBR材质、HDR光照、后期处理链
   - **互动方向**：深入研究物理引擎、复杂交互动画

3. **个人旗舰项目**
   - 整合所有技术积累
   - 撰写详尽的技术架构文档
   - 优化到工业级可用标准

4. **社区参与**
   - 积极参与Three.js等开源项目Issue讨论
   - 尝试提交第一个PR
   - 在技术社区回答相关问题

#### 平日任务
- 完善旗舰项目和文档
- 准备“高级图形工程师”简历和作品集
- 输出系列文章：《我的可视化专家成长之路》

## 📚 核心学习资源

### 必读纸质书
1. **《3D数学基础：图形与游戏开发》** - 基础中的基础
2. **《WebGL编程指南》** - WebGL“红宝书”
3. **《Real-Time Rendering 4th》** - 可作参考书，不必通读

### 核心在线课程
1. **Three.js Journey**（Bruno Simon） - 最佳Three.js实践课程
2. **Shadertoy** - 全球顶级着色器作品集，培养感觉
3. **WebGL Fundamentals**（WebGL基础教程网站）

### 关键技术栈
- **核心框架**：Three.js（主）、Babylon.js（了解）
- **数学库**：gl-matrix（手写理解后使用）
- **协议**：MQTT（物联网）、WebSocket
- **辅助库**：ECharts、D3.js、Stats.js、dat.GUI
- **工具**：Blender、VSCode、Chrome DevTools

## ⚠️ 关键注意事项

### 避免的误区
1. **不要跳过数学**：矩阵不理解，永远在抄代码
2. **不要只看不写**：图形学必须亲手敲代码，调参数
3. **不要追求速成**：这是一条陡峭的学习曲线，需要耐心
4. **不要孤立学习**：积极参与社区，关注行业动态

### 心态建设
- **享受困难**：每个遇到的难题都是你未来竞争力的壁垒
- **保持输出**：写作是最好的学习方式
- **寻找同路人**：加入可视化相关社群，交流学习
- **接受迭代**：第一个版本很丑很正常，关键是持续改进
